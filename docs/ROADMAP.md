# 개발 로드맵 (Notion CMS 블로그)

**전체 예상 기간**: 11-16일 (병렬 작업 고려 시 단축 가능)

---

## 📌 핵심 순서

개발 단계별 핵심 작업을 한눈에 보기:

1. **Phase 1: 프로젝트 초기 설정 및 Notion API 연동**
2. **Phase 2: 공통 API 함수 및 재사용 컴포넌트**
3. **Phase 3: 블로그 목록 및 상세 페이지**
4. **Phase 4: 카테고리 필터 및 검색 기능**
5. **Phase 5: SEO 최적화 및 Vercel 배포**

## Phase 1: 프로젝트 골격 (1-2일)

### 🤔 왜 이 순서인가?
**견고한 기반 없이는 기능 개발이 불가능합니다.**
- 프로젝트 구조가 없으면 개발할 곳이 없음
- 환경 변수와 API 키 설정이 선행되어야 모든 기능이 작동
- 초기 설정을 제대로 하면 이후 모든 개발이 수월해짐
- 다른 Phase는 모두 이 단계의 결과물 위에서 진행됨

### 📋 담당 작업
- [ ] Next.js 15 프로젝트 생성 및 구조 설정
- [ ] TypeScript, Tailwind CSS, shadcn/ui 초기 설정
- [ ] 환경 변수 파일 생성 (`.env.local`)
- [ ] `@notionhq/client` 패키지 설치
- [ ] Notion Integration 생성 및 API 키 획득
- [ ] 기본 레이아웃 컴포넌트 틀 생성

### ⏱️ 예상 소요 시간
- **최단**: 1일 (모든 설정이 매끄럽게 진행)
- **평균**: 1.5일
- **최장**: 2일 (Notion API 문제 해결 포함)

### ✅ 완료 기준
- `npm run dev` 실행 시 [http://localhost:3000](http://localhost:3000) 접속 가능
- Notion API 키로 데이터베이스 연동 확인
- 프로젝트 폴더 구조 완성
  ```
  src/
  ├── app/
  ├── components/
  ├── lib/
  ├── types/
  └── styles/
  ```

## Phase 2: 공통 모듈 (2-3일)

### 🤔 왜 이 순서인가?
**모든 기능에서 재사용되는 코드를 먼저 만들어야 중복을 방지합니다.**
- Notion API 호출 로직이 모든 페이지에서 필요
- 타입 정의가 없으면 기능 개발 중 타입 에러가 빈번
- 공통 컴포넌트(Header, Footer 등)는 모든 페이지에 포함
- 나중에 수정하려면 여러 파일을 동시에 변경해야 함
- 지금 완성하면 기능 개발 시 API와 타입에만 집중 가능

### 📋 담당 작업

#### 2.1 Notion API 래퍼 (`lib/notion.ts`)
- [ ] Notion 클라이언트 초기화
- [ ] 데이터베이스 쿼리 함수
  - `fetchPublishedPosts()` - 발행된 글 목록 조회
  - `fetchPostBySlug(slug)` - 특정 글 조회
  - `fetchPostsByCategory(category)` - 카테고리별 글 조회
  - `fetchPageContent(pageId)` - Notion 페이지 블록 조회
- [ ] 에러 핸들링 및 재시도 로직

#### 2.2 공통 타입 정의 (`types/notion.ts`)
- [ ] `Post` 인터페이스 (title, slug, category, tags, publishedDate, status, notionId)
- [ ] `Category` 타입
- [ ] `NotionBlock` 타입 (paragraph, heading, list, code, image, quote 등)
- [ ] API 응답 타입

#### 2.3 공통 컴포넌트
- [ ] `components/Header.tsx` - 네비게이션, 검색창
- [ ] `components/Footer.tsx` - 푸터, 저작권
- [ ] `components/PostCard.tsx` - 글 카드 (제목, 카테고리, 태그, 발행일)
- [ ] `components/Layout.tsx` - 전체 레이아웃 래퍼

### ⏱️ 예상 소요 시간
- **최단**: 2일 (API 구조가 명확함)
- **평균**: 2.5일
- **최장**: 3일 (Notion 블록 타입 복잡도)

### ✅ 완료 기준
- 모든 함수에 TypeScript 타입 정의 완료
- `lib/notion.ts` 기본 함수 구현 완료
- 공통 컴포넌트 렌더링 확인
- 단위 테스트 작성 및 통과

## Phase 3: 핵심 기능 (3-4일)

### 🤔 왜 이 순서인가?
**블로그의 존재 이유인 기능을 먼저 완성해야 합니다.**
- 글 목록과 상세 페이지가 없으면 블로그로서 기능 불가
- 사용자는 글을 읽기 위해 방문하므로 이 기능이 가장 중요
- MVP(Minimum Viable Product)를 완성하는 단계
- 이 단계에서 사용자가 실제로 블로그를 사용할 수 있음
- 추가 기능은 핵심 기능이 완성된 후에도 나중에 추가 가능

### 📋 담당 작업

#### 3.1 홈 페이지 (`app/page.tsx`)
- [ ] Notion DB에서 발행된 글 목록 조회
- [ ] 최신순으로 정렬하여 카드 렌더링
- [ ] 페이지네이션 구현 (선택적)

#### 3.2 글 상세 페이지 (`app/posts/[slug]/page.tsx`)
- [ ] 동적 라우팅 설정
- [ ] Notion 페이지 콘텐츠 조회
- [ ] 메타정보 표시 (카테고리, 태그, 발행일)

#### 3.3 Notion 렌더러 (`components/NotionRenderer.tsx`)
- [ ] 지원 블록 타입별 렌더러 구현
  - Paragraph, Heading (1~3)
  - Bulleted List, Numbered List
  - Code (문법 강조)
  - Image
  - Quote
  - Divider
  - Link
- [ ] 클라이언트 사이드 렌더링

### ⏱️ 예상 소요 시간
- **최단**: 3일 (블록 타입이 많지 않음)
- **평균**: 3.5일
- **최장**: 4일 (블록 렌더링 디버깅)

### ✅ 완료 기준
- 홈 페이지에서 최근 글 목록 표시 확인
- 글 상세 페이지에서 Notion 콘텐츠 렌더링 확인
- 모든 지원 블록 타입이 정상 표시
- 반응형 레이아웃 확인 (모바일 테스트)

## Phase 4: 추가 기능 (2-3일)

### 🤔 왜 이 순서인가?
**핵심 기능이 완성된 후에 사용자 경험을 향상시킵니다.**
- 카테고리 필터와 검색은 글이 많아야 가치가 있음
- 먼저 핵심 기능으로 충분히 테스트해서 안정성 확보
- 추가 기능이 없어도 블로그는 정상 작동
- 이 기능들은 사용자 편의성을 위한 부가 기능
- 우선순위를 정하고 필요한 것부터 추가 가능

### 📋 담당 작업

#### 4.1 카테고리 필터링 (`app/category/[category]/page.tsx`)
- [ ] 동적 라우팅 설정
- [ ] 카테고리별 글 목록 조회
- [ ] 카테고리 목록 표시 (사이드바 또는 탭)
- [ ] 선택된 카테고리 강조

#### 4.2 검색 기능
- [ ] 클라이언트 사이드 검색 구현
- [ ] 검색어 하이라이팅
- [ ] 검색 결과 표시 페이지

#### 4.3 SEO 최적화
- [ ] 메타 태그 설정 (title, description, og:image)
- [ ] sitemap.xml 생성 (선택적)
- [ ] robots.txt 설정 (선택적)

### ⏱️ 예상 소요 시간
- **최단**: 2일 (검색과 카테고리가 간단함)
- **평균**: 2.5일
- **최장**: 3일 (SEO 최적화 세부 작업)

### ✅ 완료 기준
- 카테고리 페이지에서 필터링 작동 확인
- 검색 기능으로 글 검색 가능
- 검색 결과 페이지 렌더링 확인
- 메타 태그가 브라우저 개발자 도구에서 보임

## Phase 5: 최적화 및 배포 (1-2일)

### 🤔 왜 이 순서인가?
**모든 기능이 완성되고 안정적일 때 최적화를 시작합니다.**
- 불완전한 코드를 최적화하면 나중에 다시 수정해야 함
- 배포 전에 성능 문제를 찾아서 개선해야 사용자 경험이 좋음
- 기능이 완성되어야 전체 성능을 정확히 측정 가능
- 불필요한 코드를 최적화하는 시간을 낭비하지 않기 위함
- 마지막 단계이므로 배포 직전의 모든 점검 수행

### 📋 담당 작업

#### 5.1 성능 최적화
- [ ] ISR (Incremental Static Regeneration) 설정
  - 글 목록: `revalidate: 3600` (1시간)
  - 글 상세: `revalidate: 86400` (24시간)
- [ ] 이미지 최적화 (Next.js Image 컴포넌트)
- [ ] 번들 크기 분석 및 최적화
- [ ] 캐싱 전략 수립

#### 5.2 반응형 디자인 개선
- [ ] 모바일 (sm: 640px) 테스트
- [ ] 태블릿 (md: 768px) 테스트
- [ ] 데스크탑 (lg: 1024px) 테스트
- [ ] Tailwind CSS 미디어 쿼리 조정

#### 5.3 Vercel 배포
- [ ] Vercel 프로젝트 생성
- [ ] 환경 변수 설정 (NOTION_API_KEY, NOTION_DATABASE_ID)
- [ ] 배포 및 프로덕션 테스트
- [ ] 커스텀 도메인 설정 (선택적)

### ⏱️ 예상 소요 시간
- **최단**: 1일 (배포 매끄러움)
- **평균**: 1.5일
- **최장**: 2일 (최적화 이슈 해결)

### ✅ 완료 기준
- Lighthouse 성능 점수 90점 이상
- First Contentful Paint (FCP) < 1.5초
- Vercel에서 프로덕션 배포 완료
- 모든 라우트 정상 작동 (모바일, 데스크탑)
- TTFB (Time to First Byte) < 200ms
